# Spec-driven development

- **spec-writing** — behavior-focused spec for a new feature.
- **review-requirements** — EARS compliance, completeness, consistency.
- **review-design** — coverage, feasibility, architectural soundness.
- **review-plan** — coverage against requirements and design, no ambiguities.
